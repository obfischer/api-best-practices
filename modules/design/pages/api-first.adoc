=== Formale Validierung möglich

Eine formale Spezifikation einer API mittels einer für den Typ der API vorgesehenen Beschreibungssprache, beispielsweise mittels der OpenAPI Specification für REST-APIs oder der ganz alten Web Services Description Language (WDSL) für SOAP Services, resultiert immer in einer maschinenlesbaren Beschreibung der API.

Deren Existenz erlaubt es, die Frage zu stellen und mittels Toolings zu beantworten: "Ist die Spezifikation syntaktisch korrekt?"

Auch die darüber hinausgehende Frage "Entspricht die API unseren eigenen Standards" kann so beantwortet und die Einhaltung sichergestellt werden.
