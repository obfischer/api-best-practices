== Wesentliche Eigenschaften

* Vertrag, der verbindlich ist
* Zentrale Spezifikation, die geteilt werden kann
* Kommunikation über Spezifikaiton
* API ist First Class Citizen


== Vorteile

- Entkopplung, interna werden von der Schnittstelle abgetrennt
- Bessere Interoperabilität und Integration
- Implizite Dokumentation
- Formale Validierung möglich
- Nachhaltig
- Aufwand für Reverse Engineering sinkt
- Effizienz steigt, da Teams parallel die API implenentieren können

== Nachteile

- Die Implementierung muss sich an Spezifikation halten

== Übliche Missverständnisse

- Design muss nicht von Implementierung entkoppelt werden
- Das ist nicht agil
- Das ist zu aufwendig

== Best Practices

- Klarheit über Datenmodel
- Prototypische Implementierung parallel zur Spezifikation
- Formale Validierung!
